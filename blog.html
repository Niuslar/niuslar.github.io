HIL with Raspberry-Pi and STM32 Nucleo-64

May 17, 2022 | Link to Github Repository

After several months of working on personal projects, I decided it was time to contribute to a bigger project and work as part of a team. That’s when I found a small team of engineers who let me join their team and help in developing an automatic incubator for laboratories. Working with experienced engineers was exactly what I needed to improve my skills as an embedded engineer, and here’s a summary of my experience for the past five months.

The goal of the project

Working with experienced engineers helped me realise what I was doing right and wrong and how I could improve as an embedded engineer. During the course of this project, I have learnt from using Git/Github as part of a team, to solving problems in a more efficient way.

Here are some of my main contributions to the project

UART Communication

After writing the code to read ADC inputs and convert these to temperature, one important feature that we needed was to communicate with the MCU. This could be used to send messages from the MCU to a terminal and to send commands to the MCU for debugging.

In order to achieve this, I had to create the following:

  • FIFO Buffer to store incoming/outgoing data
  • JSON-based parser to process commands

    • FIFO Buffer

    While C++ has std::queues to implement FIFO buffers easily, I wanted to write my own template class as a learning experience. One of the main problems I faced with this buffer was an error in the data received when I sent data via UART very quickly. This was caused by a simultaneous access to a buffer from an interrupt handler and from the main program. To solve this, I opted to disable interrupts while the buffer is putting data into the queue.
